# 高校野球 通算ランキング

高校野球(甲子園・都道府県大会)の戦績データベースサイト。
公開URL: https://koshien-ranking.com (Cloudflare Workers、pushで自動デプロイ)

## サイト構成

- `/` … トップ。甲子園(春・夏総合)の**対話型ランキングアプリ**(埼玉と同じUI・同じ算定方法)
  + その下にタイル型日本地図(都道府県大会へのリンク)
- `/saitama/` … 埼玉大会の対話型ランキングアプリ
- `/saitama/schools/` `/saitama/years/` … 埼玉の学校別・年度別ページ(自動生成)
- 他の都道府県も `data/<slug>/results.csv` を置けば同じ構成で自動生成される
- **北海道は北北海道(kitahokkaido)・南北海道(minamihokkaido)に分割**して別都道府県として扱う(計48)

## データの更新方法

1. **都道府県大会**: `data/<県slug>/results.csv` を編集(UTF-8)
   - 列: `年,ブロック,優勝,準優勝,ベスト4,ベスト4,ベスト8,ベスト8,ベスト8,ベスト8,決勝勝者得点,決勝敗者得点`
   - ブロック列は通常空欄。ブロック制の年のみ `A`/`B`/`東`/`西`/`南`/`北` を入れる
   - スコア詳細は `data/<県slug>/scores.json`(キーは `"年|ブロック"`)
   - **ランキング計算のパラメータ**: `data/<県slug>/config.csv`(キー,値,説明)。
     到達点(pCH/pRU/pB4/pB8)、点差上限(cap)、継続係数(wc)・連覇加算(streak)、
     格上撃破の係数(uu)・上限(ucap)、除外年(excluded、複数はセミコロン区切り)
   - **校名の統合(改名対応)**: `data/<県slug>/aliases.csv`(統合前,統合後)
   - アプリの「データ入力」「設定」タブは廃止済み。データ・設定の変更はすべて上記CSVの編集→ビルドで行う
2. **甲子園(トップのアプリ)**: `data/koshien/results.csv` を編集する
   - **自動抽出**: `tools/` に出典サイト(bibijr.vivian.jp)からの取得・パース・生成スクリプトがある。
     新年度追加時は `fetch_rest.py` で新ページを取得 → `integrate.py` で results.csv / scores.json を再生成
     (integrate.py 内のパス定数をリポジトリに合わせて実行。取得は1ページ0.7秒間隔で丁寧に)
   - **都道府県予選の自動抽出**: `tools/pref_fetch.py <県ローマ字>` で取得 →
     `tools/pref_integrate.py <県ローマ字> <県名漢字>` で results.csv / scores.json を生成(千葉・神奈川で使用済み)。
     URLは `bibijr.vivian.jp/<県ローマ字>/<選手権の回次>`(分割年は末尾 e/w/n/s)。
     合同予選(南関東・京浜・神静等)の年は大会名で自動除外。不戦勝で欠けた大会はスコア詳細を
     出力せず成績のみモードに落とす(全スコアが揃わないと大会ごと未完成扱いになるため)
   - 列: `年,大会,優勝,準優勝,ベスト4,ベスト4,ベスト8,ベスト8,ベスト8,ベスト8,決勝勝者得点,決勝敗者得点`
   - 「大会」列は `春` / `夏`。学校名は「校名(都道府県)」形式で記録(アプリ注入時に自動で県を外して名寄せ)
   - **学校名は大会当時の名称で記録する**。改名前後の同一校は `data/koshien/aliases.csv` に
     「旧名,現名」を登録する → ランキング内部では同一校として合算し、最新の校名で表示される
   - パラメータは `data/koshien/config.csv`、出典表記は `data/koshien/sources.txt`
   - 準々決勝以降の全スコアを `data/koshien/scores.json` に置けば点差ボーナスが全試合で効く(現状は決勝のみ)
3. ビルド: `python build.py`(全ページ・sitemap再生成)
4. 反映: `git add -A` → `git commit` → `git push`

県slug一覧・地図タイル座標は build.py の `PREFS` に定義。

## ファイル構成

| パス | 内容 | 手で編集する? |
|---|---|---|
| `data/saitama/results.csv` ほか | データの原本 | **する** |
| `build.py` | サイト生成スクリプト(都道府県定義もここ) | 機能追加時のみ |
| `app_template.html` | 対話型アプリのテンプレート | しない |
| `index.html` `<県slug>/` `sitemap.xml` | 生成物 | **しない(build.pyが生成)** |
| `about.html` ほか固定ページ | サイト情報・ポリシー類 | 必要に応じて |
| `prototype.html` | 初期の原本(保存用、配信されない) | しない |
